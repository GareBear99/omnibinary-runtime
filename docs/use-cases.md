# Use Cases

## What a user can do now

- inspect a target file and learn what it likely is
- understand whether the target can run natively, is preview-only, or is currently blocked
- launch a compatible native target through the proof path
- generate receipts and reports for engineering review

## What a maintainer can do now

- audit repo readiness
- inspect blocker and backlog reports
- validate cache/report integrity
- generate evidence packs and readiness summaries
- prepare the repo for the first real execution milestone

## What the future product should do

- accept a supported foreign-ISA target
- decode guest instructions
- lower them into canonical IR
- emit host blocks
- reuse translated blocks safely
- mediate syscalls/personality rules
- isolate execution safely
